# SportTelemetry

## Разработка прототипа системы телеметрии для спортивной медицины

### Описание проекта

**SportTelemetry** — программное средство, предназначенное для сбора, обработки, хранения и визуализации телеметрических данных спортсменов. Система позволяет анализировать показатели частоты сердечных сокращений (ЧСС), уровень физической нагрузки и степень восстановления спортсмена в процессе тренировочной или соревновательной деятельности.

Проект разработан в рамках курсовой работы по дисциплине **«Технология программирования»**.

# 1. Назначение программного средства

Программное обеспечение предназначено для автоматизации процессов:

* сбора телеметрических данных;
* обработки временных рядов;
* анализа частоты сердечных сокращений;
* определения зон интенсивности нагрузки;
* визуализации спортивных показателей;
* хранения результатов тренировочных сессий.

Система может использоваться:

* спортивными врачами;
* тренерами;
* аналитиками;
* исследовательскими лабораториями;
* студентами спортивных и технических направлений подготовки.

# 2. Основные возможности

## Импорт данных

Поддерживаются входные файлы форматов:

* `.txt`
* `.log`

Во время загрузки выполняется:

* чтение данных;
* синтаксический анализ;
* проверка корректности значений;
* фильтрация аномальных показателей.

## Анализ телеметрии

Программа выполняет:

* расчет средней ЧСС;
* расчет максимальной ЧСС;
* определение зон интенсивности;
* анализ физической нагрузки;
* анализ восстановления спортсмена.

## Визуализация данных

Реализовано отображение:

* графика ЧСС;
* временной шкалы тренировки;
* зон интенсивности;
* показателей нагрузки и восстановления.

Графики поддерживают:

* масштабирование;
* прокрутку;
* интерактивное отображение данных.

## Архивирование

Система позволяет:

* сохранять завершенные сессии;
* просматривать архив тренировок;
* выполнять повторный анализ ранее сохраненных данных.

# 3. Архитектура приложения

Проект реализован в соответствии с архитектурным шаблоном **MVC (Model–View–Controller)**.

## Model

Содержит бизнес-логику приложения.

Основные задачи:

* хранение телеметрических данных;
* расчет статистических показателей;
* определение пульсовых зон;
* сериализация данных.

### Основные классы

#### TelemetryModel

Хранит:

* временные ряды ЧСС;
* данные нагрузки;
* показатели восстановления;
* статистику текущей сессии.

#### SessionResult

Представляет завершенную тренировочную сессию и содержит:

* имя спортсмена;
* параметры сессии;
* статистические показатели;
* сериализованные данные.

---

## View

Отвечает за графический интерфейс пользователя.

Реализован средствами:

* JavaFX;
* FXML;
* CSS.

Интерфейс включает:

* область импорта данных;
* область визуализации;
* панель статистики;
* архив тренировок.

---

## Controller

Обеспечивает взаимодействие между интерфейсом и моделью данных.

Функции контроллера:

* обработка действий пользователя;
* запуск вычислений;
* обновление графиков;
* сохранение результатов.

# 8. Описание модели данных

## TelemetryModel

Основная модель приложения содержит:

* данные ЧСС;
* данные нагрузки;
* данные восстановления;
* результаты вычислений.

### Выполняемые операции

* хранение временных рядов;
* вычисление средней ЧСС;
* вычисление максимальной ЧСС;
* определение зон интенсивности.

---

## SessionResult

Используется для хранения завершенных тренировок.

Содержит:

* имя спортсмена;
* название сессии;
* среднюю ЧСС;
* максимальную ЧСС;
* определенную зону нагрузки;
* сериализованные данные тренировки.

# 9. Тестирование

Для проверки корректности работы системы используется библиотека **JUnit 5**.

Проверяются:

* создание объектов модели;
* создание объектов SessionResult;
* корректность пользовательских данных;
* работоспособность бизнес-логики.

Запуск тестов:

```bash
mvn test
```
