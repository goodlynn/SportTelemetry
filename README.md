# Прототип локальной системы телеметрии для спортивной медицины

## Введение

**Актуальность проблемы.** В современной спортивной медицине и профессиональном тренировочном процессе критически важную роль играет оперативный мониторинг функционального состояния организма спортсмена. Одним из наиболее информативных показателей реакций сердечно-сосудистой системы на физическую нагрузку является частота сердечных сокращений (ЧСС). Непрерывный контроль пульса в реальном времени, а также автоматизированный расчет уровней физиологической нагрузки и стадий восстановления позволяют спортивным врачам и тренерам своевременно корректировать интенсивность тренировок, предотвращать перетренированность и минимизировать риски развития патологических изменений в миокарде. Разработка программного обеспечения для агрегации и графической визуализации телеметрических данных представляет собой актуальную научно-практическую задачу.

В связи с этим была сформулирована цель курсового проекта: разработать прототип локальной системы телеметрии для спортивной медицины, предназначенный для автоматизированного экспресс-мониторинга, анализа и интерактивной визуализации показателей частоты сердечных сокращений, уровня физиологической нагрузки и резерва восстановления спортсмена в реальном времени.

Для достижения поставленной цели необходимо решить следующие задачи:
* определить и формализовать сценарии использования программного средства в условиях симуляции тренировочного процесса;
* спроектировать архитектуру программного обеспечения и адаптивный пользовательский интерфейс графического монитора;
* реализовать и протестировать прототип системы с использованием современных технологий объектно-ориентированного программирования.

## 1. Функциональное назначение программного средства

### 1.1 Классы решаемых задач
Разрабатываемое программное средство относится к классу информационно-аналитических графических систем реального времени. В соответствии с декомпозицией бизнес-целей, система обеспечивает решение следующих классов задач:
* **задачи поиска:** оперативное нахождение критических отклонений и пиковых значений частоты сердечных сокращений в рамках текущей запущенной сессии мониторинга;
* **задачи анализа (проверки и валидации):** непрерывная обработка входящих телеметрических сигналов, автоматическая классификация текущего пульса по пяти клиническим терапевтическим зонам интенсивности, а также расчет интегральных показателей нагрузки и восстановления;
* **задачи оптимизации трудозатрат при решении задач поиска и анализа:** автоматизация рутинных процессов визуализации данных, мгновенное построение динамических графиков и автоматическое вычисление параметров состояния организма без необходимости ручной фиксации показателей.

Концепция программы представлена в приложении 1. Анализ прототипов и детальное сравнение разработанного прототипа с существующими коммерческими аналогами приведены в таблице 1.

### 1.2 Функциональное назначение
Программа предназначена для решения задач автоматизации оперативного медицинского контроля в спортивных секциях, диагностических центрах и на тренировочных сборах. 

Пользователь имеет возможность осуществлять ввод личных данных спортсмена перед началом тестирования, запустить интерактивную сессию мониторинга телеметрии в реальном времени, проанализировать текущую зону интенсивности на основе цветовой индикации, получить результаты анализа и обработки данных в виде графических индикаторов нагрузки и восстановления, а также построить динамический график ЧСС со скользящим временным окном.

Сценарии использования «Интерактивный мониторинг тренировочной сессии» представлены в приложении 2.

### 1.3 Нефункциональные требования, ограничения и интерфейсы
В рамках проектирования программного комплекса были зафиксированы строгие технические требования. В области надежности система обязана корректно обрабатывать изменения пульса и исключать перегрузку оперативной памяти при длительной отрисовке за счет автоматического удаления устаревших узлов данных из графического компонента. С точки зрения интерфейса наложено ограничение на обязательное использование контейнеров панели прокрутки `ScrollPane` для предотвращения скрытия информации при изменении размеров графического окна. Физическая работа программы ограничена исполнением в среде Java Runtime Environment версии 17 или выше с поддержкой модулей графической библиотеки JavaFX.

Техническое задание представлено в приложении 3.

## 2. Проектная часть

### 2.1. Архитектура приложения
Для обеспечения простоты сопровождения, читаемости кода и удобства масштабирования проекта была выбрана архитектура MVC (Model-View-Controller). Выбор данной архитектуры обусловлен необходимостью разделения логики математических расчетов от графического представления, что позволяет модифицировать интерфейс без изменения алгоритмов анализа.

Основными компонентами приложения являются три взаимодействующих слоя в соответствии с распределением функциональности по компонентам: слой представления (View) детерминирован файлами разметки интерфейса FXML и каскадными таблицами стилей `styles.css`; слой контроллеров (Controller) инкапсулирован в классе `TelemetryController.java`, управляющем захватом событий от кнопок; слой модели данных (Model) содержит внутренние свойства с поддержкой механизмов JavaFX Data Binding для автоматического обновления графических элементов.

### 2.2. Проектирование пользовательского интерфейса
В соответствии с ТЗ интерфейс пользователя реализован в виде интерактивной графической формы ввода данных и диалога (Dashboard). Центральное поле формы отведено под компонент `LineChart`, отображающий график ЧСС в реальном времени. Справа располагается аналитическая панель, помещенная в контейнер `ScrollPane` для обеспечения вертикальной прокрутки элементов. Данная панель содержит карточки терапевтических зон ЧСС с цветовой кодировкой, а также графические прогресс-бары для визуализации текущего уровня накопленной физической нагрузки и расчетного резерва восстановления организма.

Раскадровка интерфейса представлена на рисунке 2.

### 2.3. Модель данных ПС
Основными сущностями в рамках разрабатываемого прототипа являются текущее физиологическое состояние спортсмена и точка измерения. Вся обработка информации происходит в оперативной памяти приложения. Для их описания используются наблюдаемые свойства JavaFX (`SimpleIntegerProperty` и `SimpleDoubleProperty`). К основным атрибутам сущностей относятся текущее значение частоты сердечных сокращений в ударах в минуту (BPM), уровень накопленной физической нагрузки (в диапазоне от 0.0 до 1.0), текущий текстовый статус терапевтической зоны и относительный коэффициент резерва восстановления организма.

Модель данных в виде диаграммы классов представлена на рисунке 3.

### 2.4. Моделирование работы приложения
На рисунках 4–5 представлены диаграммы последовательности, которые отображают полное взаимодействие пользователя с системой. Построение диаграмм последовательности велось вокруг центрального сценария интерактивной симуляции потока телеметрических данных. При активации пользователем кнопки «Старт» контроллер инициализирует внутренний многопоточный таймер `Timeline`. Данный таймер с периодичностью в одну секунду генерирует новые значения ЧСС, имитируя сигналы с датчиков, и передает их в модель, которая производит математический пересчет зон интенсивности, нагрузки и восстановления, мгновенно обновляя интерфейс через механизм связывания.

Управление зависимостями и сборка осуществляются автоматизированным инструментом Maven (файл `pom.xml`).

### 3.3 Создание документации
Код программного проекта загружен в репозиторий с описанием программы, выполненным по ГОСТ 19.402-78, включающим инструкцию по установке.

**Инструкция по установке и развертыванию:**
1. Установить комплект разработчика JDK версии 17 или выше и настроить системные переменные окружения (`JAVA_HOME`).
2. Разархивировать исходный код проекта в рабочую директорию на локальном компьютере или клонировать репозиторий.
3. Открыть папку с проектом в интегрированной среде IntelliJ IDEA в качестве Maven-проекта.
4. Дождаться автоматической загрузки графических зависимостей JavaFX из конфигурационного файла `pom.xml`.
5. Запустить сборку и выполнение графического приложения командой Maven `mvn clean javafx:run` или через непосредственный запуск метода `main` в классе `MainApp.java`.

## Заключение

В ходе выполнения курсового проекта был успешно разработан программный продукт «Прототип локальной системы телеметрии для спортивной медицины». 

В рамках реализации данного прототипа были успешно выполнены функции построения адаптивного графического интерфейса диагностического монитора (Dashboard) с динамической прокруткой областей индикации для предотвращения скрытия информации, интеграции симулятора медицинских датчиков пульса на базе многопоточного таймера JavaFX `Timeline`, а также разработки алгоритмов автоматического расчета физиологической нагрузки, резервов восстановления и классификации текущего пульса по пяти клиническим терапевтическим зонам ЧСС. Разработанное программное средство функционирует полностью в оперативной памяти и демонстрирует стабильность работы графической подсистемы.
