# Разработка прототипа системы телеметрии для спортивной медицины

## Введение

### Актуальность проблемы
В современной спортивной медицине и профессиональном тренировочном процессе критически важную роль играет оперативный мониторинг функционального состояния организма спортсмена. Одним из наиболее информативных показателей реакций сердечно-сосудистой системы на физическую нагрузку является частота сердечных сокращений (ЧСС). Непрерывный контроль пульса в реальном времени, а также автоматизированный расчет уровней физической нагрузки и стадий восстановления позволяют спортивным врачам и тренерам своевременно корректировать интенсивность тренировок, предотвращать перетренированность и минимизировать риски развития патологических изменений. Разработка локального программного обеспечения для агрегации и визуализации телеметрических данных является актуальной инженерной задачей.

В связи с этим была сформулирована цель курсового проекта:
Разработать **прототип локальной информационной системы телеметрии** для **автоматизированного мониторинга, анализа и визуализации показателей частоты сердечных сокращений, уровня физиологической нагрузки и резерва восстановления спортсменов в реальном времени**.

Для достижения поставленной цели необходимо решить следующие задачи:
* Определить и формализовать сценарии использования программного средства в условиях тренировочного процесса и клинического анализа.
* Спроектировать архитектуру программного обеспечения и адаптивный пользовательский интерфейс диагностического монитора.
* Реализовать и протестировать интерактивный прототип системы с использованием объектно-ориентированных технологий и многопоточных механизмов JavaFX.
* Создать сопутствующую техническую документацию и пользовательские инструкции по развертыванию.

## 1. Функциональное назначение программного средства

### 1.1 Классы решаемых задач
Программное средство относится к классу информационно-аналитических медицинских систем реального времени и решает задачи анализа (проверки и валидации), обеспечивая непрерывную валидацию поступающих телеметрических данных, автоматическую классификацию текущего пульса по пяти клиническим терапевтическим зонам интенсивности, а также динамический расчет интегральных показателей физиологической нагрузки и остаточного резерва восстановления организма спортсмена.

Кроме того, система решает задачи оптимизации трудозатрат при решении задач анализа. Автоматизация рутинных процессов регистрации физиологических показателей, мгновенное построение интерактивных графиков и автоматическое вычисление текущих зон ЧСС полностью освобождают спортивного медика от необходимости ручного пересчета параметров.

Концепция программы представлена в приложении 1. Анализ прототипов (сравнение с существующими коммерческими системами) приведён в таблице 1.

**Таблица 1. Анализ прототипов систем спортивной телеметрии**
| Критерий сравнения | Разрабатываемый прототип | Коммерческие аналоги (Polar Flow / Garmin Connect) |
| :--- | :--- | :--- |
| **Доступность исходного кода** | Открытый (Open-Source, JavaFX) | Закрытый (Proprietary) |
| **Хранение и обработка** | Локальная оперативная память (In-Memory) | Облачные проприетарные серверы (Cloud-first) |
| **Интерфейс** | Специализированный Dashboard спортивного врача | Персональный атлетический/Мобильный интерфейс |
| **Автономия работы** | Полная, независимость от внешних API сетей | Требует постоянного интернет-соединения |

### 1.2 Функциональное назначение
Программа предназначена для решения задач автоматизации медицинского контроля в спортивных секциях, диагностических центрах и на тренировочных сборах.

В соответствии с бизнес-требованиями пользователь (спортивный врач или тренер) имеет возможность вводить личные данные спортсмена (ФИО) перед началом диагностического тестирования, запускать сессию мониторинга телеметрии в реальном времени нажатием кнопки «Старт», визуализировать динамику ЧСС на интерактивном графике со скользящим временным окном в 30 секунд, проанализировать в реальном времени текущую пульсовую зону на основе цветовой индикации и текстовых подсказок, осуществлять остановку записи сессии с возможностью фиксации результатов, а также получать итоговые результаты анализа в виде графических и числовых индикаторов.

Сценарии использования «Мониторинг тренировочной сессии реального времени» представлены в приложении 2.

### 1.3 Нефункциональные требования, ограничения и интерфейсы
В рамках требований к надежности система исключает утечки оперативной памяти при длительной отрисовке графика за счет автоматического удаления устаревших узлов данных и сдвига окна в 30 секунд. Графический пользовательский интерфейс спроектирован как адаптивный, правый блок медицинских индикаторов снабжен контейнером панели прокрутки `ScrollPane` для предотвращения скрытия информации при изменении размеров окна пользователем. Существуют технические ограничения: выполнение программы осуществляется локально в оперативной памяти (In-Memory), а запуск требует среды исполнения Java Runtime Environment (JRE) версии 17 или выше с поддержкой графических модулей JavaFX.

Техническое задание представлено в приложении 3.

## 2. Проектная часть

### 2.1. Архитектура приложения
Для обеспечения модульности кода, простоты сопровождения, читаемости и удобства масштабирования проекта была выбрана архитектура **MVC (Model-View-Controller)** совместно с принципами объектно-ориентированного разделения обязанностей. Разделение системы на независимые компоненты позволяет изменять графическое представление без необходимости переписывать внутренние алгоритмы медицинских расчетов и логику генерации данных.

Основными компонентами приложения являются слой представления (файл разметки `main_view.fxml` и файл стилей `styles.css`), слой контроллеров (класс `TelemetryController.java`, осуществляющий обработку событий от элементов интерфейса и координирующий обновление GUI) и слой модели данных (класс `TelemetryModel.java`, инкапсулирующий текущие свойства ЧСС, нагрузки и восстановления с поддержкой механизма `Data Binding`).

### 2.2. Проектирование пользовательского интерфейса
В соответствии с техническим заданием интерфейс пользователя реализован в виде интерактивной графической формы управления и визуализации данных (двухпанельный диагностический Dashboard). Центральная часть отведена под графический компонент `LineChart`, а правая боковая аналитическая панель помещена в контейнер `ScrollPane` для вертикальной прокрутки карточек пульсовых зон, прогресс-баров накопленной усталости (`loadBar`) и резерва восстановления (`recoveryBar`).

Раскадровка интерфейса представлена на рис. 2.

### 3.3 Создание документации
Код программного проекта загружен в удаленный репозиторий GitHub: [https://github.com/goodlynn/SportTelemetry](https://github.com/goodlynn/SportTelemetry). Описание программы выполнено в соответствии со стандартами ЕСПД (ГОСТ 19.402-78).

#### Инструкция по установке и развертыванию:
1. Установить комплект разработчика Java Development Kit (JDK) версии 17 или выше и настроить системные переменные окружения (`JAVA_HOME`).
2. Клонировать репозиторий с проектом с помощью команды `git clone https://github.com/goodlynn/SportTelemetry.git` или разархивировать исходный код в рабочую директорию.
3. Открыть папку с проектом в интегрированной среде IntelliJ IDEA в качестве Maven-проекта.
4. Дождаться автоматической загрузки графических зависимостей JavaFX из конфигурационного файла `pom.xml`.
5. Запустить сборку и выполнение графического приложения командой Maven `mvn clean javafx:run` или через непосредственный запуск метода `main` в классе `MainApp.java`.

Разработанное руководство пользователя представлено в приложении 7.

## Заключение

В ходе выполнения курсового проекта был успешно разработан программный продукт **«Прототип локальной системы телеметрии для спортивной медицины»**. 

В рамках реализации данного прототипа были успешно выполнены функции построения адаптивного графического интерфейса диагностического монитора (Dashboard) с динамической вертикальной прокруткой (`ScrollPane`) областей индикации для исключения рисков скрытия информации, интеграции симулятора медицинских датчиков пульса на базе многопоточного таймера `JavaFX Timeline`, а также разработки алгоритмов автоматического расчета физиологической нагрузки, резервов восстановления и классификации текущего пульса по пяти клиническим терапевтическим зонам ЧСС. Разработанное программное средство функционирует стабильно, выполняет обработку полностью в оперативной памяти и отвечает всем поставленным требованиям к интерактивным прототипам медицинских мониторов.
